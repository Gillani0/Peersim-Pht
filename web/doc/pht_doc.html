We give here a brief summary of a Prefix hash tree (Pht). For a more precise informations, please read the original paper.

Data Structure

Prefix hash tree is a distributed data structure that relies on a Distributed hash table (Dht). Every node of the tree is labeled with a binary string, e.g.: the sons of a node whose label is l will be l0 and l1. It has the following properties:

  1. (Universal prefix) Each node has either 0 or 2 children.
  2. (Keystorage) A key K is stored at a leaf node whose label is a prefix of K.
  3. (Split) Each leaf node stores atmost B keys.
  4. (Merge) Each internal node contains at least (B+1) keys in its sub-tree.
  5. (Threadedleaves) Each leaf node maintains a pointer to the leaf nodes on its immediate left and immediate right respectively.

Operations

A Pht provides some basic and more complex operations. The basic ones are:

  1. Insert: insert a key with its data into the Pht.
  2. Delete: delte the given key with its related data from the Pht.
  3. Binary lookup: try different prefix of the key until the leaf whose label is a prefix a the key, is founded.
  4. Linear Lookup: start a lookup from the root node.

The two more complex operations are range queries. To work they need a to use one of the two possible lookups.

  1. Sequential Query: search for the leaf which is responsible for the beginning of the range key, then use the links between leaves to visit every leaf that has key in the searched range.
  2. Parallel Query: search for the node whose label is the smallest common prefix between the beginning and the end of the range keys, and flood its sub-tree (the leaves in the range will send their keys).
